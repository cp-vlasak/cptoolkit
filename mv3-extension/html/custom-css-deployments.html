How to Use Custom CSS Deployments

What is this?

Custom CSS Deployments lets you create reusable CSS rules that automatically inject into CivicPlus CMS pages when they load. Use it to tweak admin UI styling, fix layout quirks, or apply consistent visual overrides across pages.

Creating a Rule

  1. Click New Rule and give it a descriptive name.
  2. Choose a Scope:
    • Admin Pages Only (recommended) — only applies on /Admin/ paths. Prevents accidental changes to public-facing pages.
    • Any Page on This Site — applies everywhere on the domain, including the live site front.
  3. Pick a Page Match Type and enter a Match Value to target specific pages.
  4. Write your CSS in the editor (or click Load CSS File to import a .css file).
  5. Click Save Rule.

Page Match Types

TypeBehaviorExample
Path Starts With Matches if the URL path begins with the value /Admin/DesignCenter/Themes
Exact Path Matches only if the path is exactly the value /Admin/DesignCenter/Layouts
Path Contains Matches if the path includes the value anywhere /DesignCenter/
URL Contains Matches against the full URL (including query string) themeID=1
Regex on Path Matches the path against a regular expression ^/Admin/DesignCenter/(Themes|Layouts)

Managing Rules

  • Enable/Disable — toggle a rule without deleting it. Disabled rules are saved but won't inject CSS.
  • Duplicate — create a copy of an existing rule to use as a starting point.
  • Export/Import JSON — share rules with teammates or back them up. You can export individual rules or all rules at once.
  • Import CSS File — quickly create a new rule from a .css file. You'll be prompted to set the page match before saving.

Tips

  • Rules are stored locally in your browser. They are not synced to other machines unless you export and import the JSON.
  • Multiple rules can match the same page — all matching CSS is injected in rule-list order.
  • If the tool is disabled in the popup toggle, no rules will deploy until it is re-enabled.
  • CSS is injected as a <style> tag into the page, so standard CSS specificity applies. Use more specific selectors or !important if your styles aren't taking effect.
